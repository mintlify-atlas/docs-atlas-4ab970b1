This guide will help you pack your first repository with Repomix and use the output with AI assistants.

## Pack your first repository

<Steps>
  <Step title="Navigate to your project">
    Open a terminal and navigate to your project directory:
    
    ```bash
    cd /path/to/your/project
    ```
  </Step>
  
  <Step title="Run Repomix">
    Run Repomix to pack your entire repository:
    
    ```bash
    npx repomix@latest
    ```
    
    <Tip>
    Using `npx repomix@latest` ensures you always use the most recent version without installing anything.
    </Tip>
    
    You'll see output like this:
    
    ```
    📦 Repomix v1.12.0

    ✔ Repository files collected
    ✔ Security check completed
    ✔ File contents processed
    ✔ Output file generated
    
    📊 Total Files: 42
    📊 Total Characters: 125,430
    📊 Total Tokens: 31,250
    
    ✅ Output file: repomix-output.xml
    ```
  </Step>
  
  <Step title="Find your output file">
    Repomix creates a file called `repomix-output.xml` in your current directory. This file contains your entire codebase in an AI-friendly format.
  </Step>
</Steps>

## Use output with AI assistants

Now that you have your packed repository, here's how to use it with popular AI tools:

<Tabs>
  <Tab title="Claude / ChatGPT">
    <Steps>
      <Step title="Open the AI chat interface">
        Navigate to [Claude](https://claude.ai) or [ChatGPT](https://chat.openai.com).
      </Step>
      
      <Step title="Attach the output file">
        Click the attachment button (📎) and upload `repomix-output.xml`.
      </Step>
      
      <Step title="Add your prompt">
        Send a prompt along with the file:
        
        ```
        This file contains all the files in the repository combined into one.
        I want to refactor the code, so please review it first.
        ```
      </Step>
      
      <Step title="Get AI analysis">
        The AI will analyze your codebase and provide insights, suggestions, or generate code based on your request.
      </Step>
    </Steps>
  </Tab>
  
  <Tab title="Copy to clipboard">
    <Steps>
      <Step title="Run with --copy flag">
        Generate output and copy to clipboard in one command:
        
        ```bash
        npx repomix@latest --copy
        ```
      </Step>
      
      <Step title="Paste into AI chat">
        Paste directly into any AI chat interface using Cmd+V (Mac) or Ctrl+V (Windows/Linux).
      </Step>
    </Steps>
    
    <Note>
    The `--copy` flag automatically copies the output to your system clipboard after generation.
    </Note>
  </Tab>
  
  <Tab title="Command line">
    <Steps>
      <Step title="Output to stdout">
        Generate output directly to stdout and pipe to another tool:
        
        ```bash
        npx repomix@latest --stdout | llm "Please explain what this code does."
        ```
      </Step>
      
      <Step title="Use with other CLI tools">
        Combine with tools like [simonw/llm](https://github.com/simonw/llm) for command-line AI workflows:
        
        ```bash
        # Analyze architecture
        npx repomix@latest --stdout | llm "Describe the overall architecture"
        
        # Generate documentation
        npx repomix@latest --stdout | llm "Create a README.md" > README.md
        
        # Find security issues
        npx repomix@latest --stdout | llm "Check for security vulnerabilities"
        ```
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Example AI prompts

Once you've shared your codebase with an AI assistant, try these prompts:

### Code review

```
This file contains my entire codebase. Please review the overall structure 
and suggest any improvements or refactoring opportunities, focusing on 
maintainability and scalability.
```

### Generate documentation

```
Based on the codebase in this file, please generate a detailed README.md 
that includes an overview of the project, its main features, setup 
instructions, and usage examples.
```

### Create tests

```
Analyze the code in this file and suggest a comprehensive set of unit tests 
for the main functions and classes. Include edge cases and potential error 
scenarios.
```

### Assess code quality

```
Review the codebase for adherence to coding best practices and industry 
standards. Identify areas where the code could be improved in terms of 
readability, maintainability, and efficiency.
```

## Pack specific files or directories

You don't always need to pack your entire repository. Here's how to be selective:

### Pack a specific directory

```bash
npx repomix@latest src
```

### Pack multiple directories

```bash
npx repomix@latest src tests
```

### Include specific file patterns

```bash
npx repomix@latest --include "src/**/*.ts,**/*.md"
```

This includes only TypeScript files in `src/` and all Markdown files.

### Exclude specific patterns

```bash
npx repomix@latest --ignore "**/*.test.ts,**/*.spec.ts"
```

This excludes all test files.

## Choose an output format

Repomix supports multiple output formats optimized for different use cases:

<CodeGroup>

```bash XML (default)
npx repomix@latest
# Creates: repomix-output.xml
```

```bash Markdown
npx repomix@latest --style markdown
# Creates: repomix-output.md
```

```bash JSON
npx repomix@latest --style json
# Creates: repomix-output.json
```

```bash Plain text
npx repomix@latest --style plain
# Creates: repomix-output.txt
```

</CodeGroup>

<Tip>
XML format is recommended for most AI tools as it provides better structure and context. Learn more about [why XML works well with AI](https://docs.anthropic.com/en/docs/build-with-claude/prompt-engineering/use-xml-tags).
</Tip>

## Reduce token count

If your repository is large, you might exceed AI token limits. Here are strategies to reduce token count:

### Use compression

```bash
npx repomix@latest --compress
```

This uses Tree-sitter to extract only essential code structure (classes, functions, interfaces), reducing tokens by ~70% while preserving semantic meaning.

### Remove comments

```bash
npx repomix@latest --remove-comments
```

### Remove empty lines

```bash
npx repomix@latest --remove-empty-lines
```

### Combine strategies

```bash
npx repomix@latest --compress --remove-comments --remove-empty-lines
```

### Check token counts

```bash
npx repomix@latest --token-count-tree
```

This shows a tree view with token counts per file/directory, helping you identify what to exclude:

```
🔢 Token Count Tree:
────────────────────
└── src/ (70,925 tokens)
    ├── cli/ (12,714 tokens)
    │   ├── actions/ (7,546 tokens)
    │   └── reporters/ (990 tokens)
    └── core/ (41,600 tokens)
        ├── file/ (10,098 tokens)
        └── output/ (5,808 tokens)
```

## Pack remote repositories

You can pack any public GitHub repository without cloning it:

```bash
npx repomix@latest --remote https://github.com/yamadashy/repomix
```

**Shorthand format:**
```bash
npx repomix@latest --remote yamadashy/repomix
```

**Specific branch:**
```bash
npx repomix@latest --remote yamadashy/repomix --remote-branch main
```

**Using branch URL:**
```bash
npx repomix@latest --remote https://github.com/yamadashy/repomix/tree/main
```

<Tip>
Packing remote repositories is perfect for:
- Analyzing open-source libraries before using them
- Learning from other projects
- Using existing codebases as implementation references for AI
</Tip>
