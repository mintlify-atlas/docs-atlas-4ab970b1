Repomix supports four output formats, each optimized for different use cases. This page explains each format with real examples from the codebase.

## Format Overview

<CardGroup cols={2}>
  <Card title="XML" icon="code">

  </Card>
  <Card title="Markdown" icon="markdown">

  </Card>
  <Card title="JSON" icon="brackets-curly">

## XML Format

The XML format uses semantic tags to structure the codebase, making it easy for AI models to parse.

### Standard XML

The default XML format uses Handlebars templates for human-readable output.

**Template:** `src/core/output/outputStyles/xmlStyle.ts:1`


### Parsable XML

The parsable XML format uses the fast-xml-parser library for machine-optimized output.

**Implementation:** `src/core/output/outputGenerate.ts:108`

```bash Usage
repomix --style xml --parsable-style
```

```xml Example
<?xml version="1.0"?>
<repomix>
  <file_summary>
    <!-- Generated by Repomix on 2024-03-15T10:30:00.000Z -->
    <purpose>This file contains a packed representation...</purpose>
    <file_format>The content is organized as follows:
1. This summary section
2. Repository structure
3. Repository files, each consisting of:
  - File path as an attribute
  - Full contents of the file</file_format>
    <usage_guidelines>When working with this file...</usage_guidelines>
    <notes>Binary files are not included...</notes>
  </file_summary>
  <directory_structure>src/
├── cli/
│   └── cliRun.ts
└── index.ts</directory_structure>
  <files>
    This section contains the contents of the repository's files.
    <file path="src/index.ts">export { pack } from './core/packager.js';</file>
    <file path="src/cli/cliRun.ts">import { Command } from 'commander';</file>
  </files>
</repomix>
```

<Note>
Parsable XML is optimized for machine parsing and is more compact than standard XML, but less human-readable.
</Note>

### Dynamic Code Fence Delimiters

Markdown output intelligently adjusts code fence delimiters to avoid conflicts:

**Implementation:** `src/core/output/outputGenerate.ts:55`

```typescript
const calculateMarkdownDelimiter = (files: ReadonlyArray<ProcessedFile>): string => {
  // Find the longest sequence of backticks in any file
  const maxBackticks = files
    .flatMap((file) => file.content.match(/`+/g) ?? [])
    .reduce((max, match) => Math.max(max, match.length), 0);
  
  // Use at least 3 backticks, or more if needed
  return '`'.repeat(Math.max(3, maxBackticks + 1));
};
```

This ensures code blocks are properly delimited even if files contain backticks.

```javascript Parsing Example
const fs = require('fs');
const output = JSON.parse(fs.readFileSync('repomix-output.json', 'utf8'));

// Access files
const indexContent = output.files['src/index.ts'];
console.log('Index file:', indexContent);

// Access directory structure
console.log('Structure:', output.directoryStructure);

// Access git logs if available
if (output.gitLogs) {
  output.gitLogs.forEach(commit => {
    console.log(`${commit.date}: ${commit.message}`);
  });
}
```

## Output Sections

All formats can include the following sections (controlled by configuration):

### File Summary

Provides metadata and usage guidelines. Disable with `--no-file-summary`.

### User Provided Header

Custom instructions or context added via `--instruction-file-path`.

```bash
echo "This codebase implements a CLI tool" > instructions.txt
repomix --instruction-file-path instructions.txt
```

### Directory Structure

Tree view of the repository structure. Disable with `--no-directory-structure`.

**Implementation:** `src/core/file/fileTreeGenerate.ts`

```
src/
├── cli/
│   ├── actions/
│   │   ├── defaultAction.ts
│   │   └── initAction.ts
│   └── cliRun.ts
└── index.ts
```

### Files

Processed file contents. Disable with `--no-files` (useful for structure-only output).

### Git Diffs

Working tree and staged changes. Enable with `--include-diffs`.

### Git Logs

Recent commit history. Enable with `--include-logs` or `--include-logs-count <n>`.

## Advanced Usage

### Copying to Clipboard

Output directly to clipboard instead of a file:

```bash
repomix --copy --style markdown
```

### Standard Output

Write to stdout for piping:

```bash
repomix --stdout --style json | jq '.files | keys'
```

### Split Output

Split output into multiple files by size:

```bash
# Split into ~10MB chunks
repomix --split-output 10000000

# Output: repomix-output-1.xml, repomix-output-2.xml, etc.
```

**Implementation:** `src/core/output/outputSplit.ts`

<Warning>
Split output cannot be used with `--copy`, `--stdout`, or `--skill-generate`.
</Warning>
