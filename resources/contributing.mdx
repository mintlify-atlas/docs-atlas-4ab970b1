## Getting Started

### 1. Set Up Development Environment

Clone the repository and install dependencies:

```bash
git clone https://github.com/yamadashy/repomix.git
cd repomix
npm install
```

### 2. Run Repomix Locally

Test your changes:

```bash
npm run repomix
```

### 3. Run Tests

Ensure all tests pass:

```bash
# Run all tests
npm run test

# Run tests with coverage
npm run test-coverage
```

### 4. Check Code Style

Repomix uses [Biome](https://biomejs.dev/) for linting and formatting:

```bash
npm run lint
```

## Pull Request Guidelines

Before submitting a pull request:

<Steps>
  <Step title="Run Tests">
    Ensure all tests pass:
    ```bash
    npm run test
    ```
  </Step>
  
  <Step title="Check Linting">
    Your code should adhere to the linting standards:
    ```bash
    npm run lint
    ```
  </Step>
  
  <Step title="Update Documentation">
    If you've added or changed functionality, update the relevant documentation (especially README.md)
  </Step>
  
  <Step title="Write Clear Commit Messages">
    Follow the [Conventional Commits](https://www.conventionalcommits.org/) specification
  </Step>
</Steps>

### Pull Request Checklist

When creating a pull request, make sure to:

- [ ] Run `npm run test`
- [ ] Run `npm run lint`
- [ ] Update documentation if needed
- [ ] Add tests for new features
- [ ] Write a clear PR description
- [ ] Reference related issues (e.g., "Fixes #123")

## Coding Standards

### Code Style

- Follow the existing code style in the project
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions focused and concise

### Testing

Repomix uses [Vitest](https://vitest.dev/) for testing:

```bash
# Run tests
npm run test

# Run tests in watch mode
npm run test -- --watch

# Generate coverage report
npm run test-coverage
```

### Commit Message Format

Follow the [Conventional Commits](https://www.conventionalcommits.org/) specification:

```
type(scope): Description

[optional body]

[optional footer]
```

**Types:**
- `feat`: New feature
- `fix`: Bug fix
- `docs`: Documentation changes
- `style`: Code style changes (formatting, etc.)
- `refactor`: Code refactoring
- `test`: Adding or updating tests
- `chore`: Maintenance tasks

**Examples:**
```
feat(cli): Add new --compress option

fix(security): Handle special characters in file paths

docs(readme): Update installation instructions
```

## Development Workflow

### Using Docker for Development

You can also develop using Docker:

```bash
# Build the Docker image
docker build -t repomix .

# Run the Docker container
docker run -v ./:/app -it --rm repomix
```

### Website Development

The Repomix website is built with [VitePress](https://vitepress.dev/):

```bash
# Prerequisites: Docker must be installed

# Start the website development server
npm run website

# Access at http://localhost:5173/
```

<Note>
Contributors only need to update the README.md file. Maintainers will handle website documentation updates.
</Note>

## Reporting Issues

When reporting bugs, please include:

- Repomix version (`repomix --version`)
- Node.js version (`node --version`)
- Operating system
- Steps to reproduce
- Expected behavior
- Actual behavior
- Error messages or logs
- Sample repository (if applicable)

### Issue Template Example

```markdown
## Description
Clear description of the issue

## Environment
- Repomix version: 1.12.0
- Node.js version: 20.0.0
- OS: macOS 14.0

## Steps to Reproduce
1. Run `repomix --remote user/repo`
2. ...

## Expected Behavior
What you expected to happen

## Actual Behavior
What actually happened

## Error Messages
```
paste error messages here
```
```

## Feature Requests

For feature requests:

1. Check [existing discussions](https://github.com/yamadashy/repomix/discussions) to avoid duplicates
2. Create a new discussion in the "Ideas" category
3. Clearly describe:
   - The problem you're trying to solve
   - Your proposed solution
   - Alternative solutions you've considered
   - Why this feature would benefit other users
