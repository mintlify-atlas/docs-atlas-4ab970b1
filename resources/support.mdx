## Bug Reports and Feature Requests

### GitHub Issues

For bug reports and feature requests:

<Card title="GitHub Issues" icon="bug" href="https://github.com/yamadashy/repomix/issues">
  Report bugs or request new features
</Card>

**When to create an issue:**
- You've found a bug or unexpected behavior
- You want to request a specific feature
- You've encountered an error that needs investigation

**Before creating an issue:**
1. Search existing issues to avoid duplicates
2. Check the [Troubleshooting guide](/resources/troubleshooting)
3. Try running with `--verbose` to get more information

**What to include in a bug report:**
- Repomix version (`repomix --version`)
- Node.js version (`node --version`)
- Operating system and version
- Complete error message
- Steps to reproduce the issue
- Expected vs actual behavior
- Output from `--verbose` mode (if applicable)

## Quick Support Tips

<Tip>
**Enable Verbose Mode**: When reporting issues, always include output from `repomix --verbose` for better diagnostics.
</Tip>

<Tip>
**Search First**: Check existing GitHub issues and discussions before creating new ones to avoid duplicates.
</Tip>

<Tip>
**Be Specific**: Provide specific examples, error messages, and reproduction steps when asking for help.
</Tip>

<Tip>
**Share Logs**: Include complete error messages and stack traces when reporting bugs.
</Tip>

## Platform-Specific Support

### Browser Extension Issues

For Chrome or Firefox extension issues:

- Check browser console for errors
- Verify extension is up to date
- Report issues on [GitHub](https://github.com/yamadashy/repomix/issues)

### MCP Server Issues

For Model Context Protocol integration:

- Verify MCP configuration
- Check AI assistant logs
- Join [Discord](https://discord.gg/wNYzTwZFku) for real-time help

### Docker Issues

For Docker-related problems:

- Ensure Docker is properly installed
- Check volume mounting permissions
- Review Docker logs: `docker logs <container-id>`
