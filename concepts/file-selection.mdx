Repomix provides powerful file selection capabilities through glob patterns, ignore files, and multiple filtering mechanisms. This page explains how to control which files are included in your output.

## File Selection Flow

Files are selected through a multi-stage filtering process:

```
┌──────────────────────┐
│  All Files in Repo   │
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│  Apply Include       │  --include patterns
│  Patterns            │  (default: all files)
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│  Apply Ignore        │  1. Default patterns
│  Patterns            │  2. .gitignore
└──────────┬───────────┘  3. .repomixignore
           │              4. Custom patterns
           ▼
┌──────────────────────┐
│  Filter by Size      │  maxFileSize (default: 50MB)
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│  Security Check      │  Secretlint scanning
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│  Selected Files      │
└──────────────────────┘
```

## Include Patterns

Include patterns specify which files to process using glob syntax.

### Basic Usage

```bash CLI
# Include only TypeScript files
repomix --include "**/*.ts"

# Include multiple patterns
repomix --include "src/**" --include "lib/**"

# Include specific file types
repomix --include "**/*.{ts,tsx,js,jsx}"
```

```json repomix.config.json
{
  "include": [
    "src/**",
    "lib/**",
    "**/*.{ts,tsx}"
  ]
}
```

### Glob Pattern Syntax

<Tabs>
  <Tab title="Wildcards">
    ```bash
    # Match any characters except /
    *.ts              # All .ts files in root
    src/*.ts          # All .ts files in src/
    
    # Match any characters including /
    **/*.ts           # All .ts files recursively
    src/**/*.test.ts  # All test files under src/
    ```
  </Tab>
  
  <Tab title="Braces">
    ```bash
    # Match one of the patterns
    **/*.{ts,tsx,js,jsx}  # TypeScript and JavaScript files
    src/{cli,core}/**     # Files in cli/ or core/ subdirs
    
    # Combine with wildcards
    **/*.{test,spec}.{ts,js}  # All test files
    ```
  </Tab>
  
  <Tab title="Negation">
    ```bash
    # Exclude specific patterns (not recommended, use --ignore instead)
    !**/*.test.ts     # Exclude test files
    
    # Better approach: use --ignore
    --include "**/*.ts" --ignore "**/*.test.ts"
    ```
  </Tab>
  
  <Tab title="Character Sets">
    ```bash
    # Match character ranges
    file[0-9].ts      # file0.ts through file9.ts
    [A-Z]*.ts         # Files starting with uppercase
    
    # Match specific characters
    file[abc].ts      # filea.ts, fileb.ts, filec.ts
    ```
  </Tab>
</Tabs>

### Default Behavior

**When no include patterns are specified:**

```typescript
if (includePatterns.length === 0) {
  includePatterns = ['**/*'];
}
```

All files are included by default (subject to ignore patterns).

### Pattern Escaping

Repomix automatically escapes special characters in file paths:

**Implementation:** `src/core/file/fileSearch.ts:67`

```typescript
export const escapeGlobPattern = (pattern: string): string => {
  // Escape backslashes first
  const escapedBackslashes = pattern.replace(/\\/g, '\\\\');
  // Escape special characters () and [], but NOT {}
  return escapedBackslashes.replace(/[()[\]]/g, '\\$&');
};
```

This handles directories with special characters like `src/(categories)/page.ts`.

### 3. .repomixignore Files

`.repomixignore` files use the same syntax as `.gitignore`.

```bash .repomixignore
# Ignore test files
**/*.test.ts
**/*.spec.ts

# Ignore documentation
docs/**
*.md

# Ignore configuration
*.config.js
tsconfig.json

# Comments are supported
# Blank lines are ignored
```

**Features:**
- Always enabled (cannot be disabled)
- Nested files have higher priority
- Supports all gitignore pattern syntax

**Pattern Parsing:** `src/core/file/fileSearch.ts:252`

```typescript
export const parseIgnoreContent = (content: string): string[] => {
  return content.split('\n').reduce<string[]>((acc, line) => {
    const trimmedLine = line.trim();
    if (trimmedLine && !trimmedLine.startsWith('#')) {
      acc.push(trimmedLine);
    }
    return acc;
  }, []);
};
```

### 4. .ignore Files

Fd-style `.ignore` files are also supported.

```bash
# Enable .ignore files (enabled by default)
repomix --dot-ignore

# Disable .ignore files
repomix --no-dot-ignore
```

**Priority:** Nested `.ignore` files override parent directory rules.

### 5. Custom Ignore Patterns

Add custom patterns via CLI or configuration:

```json repomix.config.json

    "customPatterns": [
      "**/*.test.ts",
      "**/*.spec.ts",
      "tests/**",

```

### Ignore Pattern Priority

Ignore patterns are combined with the following priority (all sources are merged):

**Implementation:** `src/core/file/fileSearch.ts:343`

```typescript
export const getIgnorePatterns = async (rootDir: string, config: RepomixConfigMerged) => {
  const ignorePatterns = new Set<string>();
  
  // 1. Add default patterns
  if (config.ignore.useDefaultPatterns) {
    for (const pattern of defaultIgnoreList) {
      ignorePatterns.add(pattern);
    }
  }
  
  // 2. Add repomix output file
  if (config.output.filePath) {
    ignorePatterns.add(relativeOutputPath);
  }
  
  // 3. Add custom patterns
  if (config.ignore.customPatterns) {
    for (const pattern of config.ignore.customPatterns) {
      ignorePatterns.add(pattern);
    }
  }
  
  // 4. Add .git/info/exclude patterns
  if (config.ignore.useGitignore) {
    const excludePatterns = parseGitExclude(rootDir);
    for (const pattern of excludePatterns) {
      ignorePatterns.add(pattern);
    }
  }
  
  return Array.from(ignorePatterns);
};
```

## Pattern Normalization

Repomix normalizes glob patterns for consistent behavior:

**Implementation:** `src/core/file/fileSearch.ts:81`

```typescript
export const normalizeGlobPattern = (pattern: string): string => {
  // Remove trailing slash (except for patterns ending with **/)
  if (pattern.endsWith('/') && !pattern.endsWith('**/')) {
    return pattern.slice(0, -1);
  }
  
  // Convert **/folder to **/folder/** for consistent ignore behavior
  if (pattern.startsWith('**/') && !pattern.includes('/**')) {
    return `${pattern}/**`;
  }
  
  return pattern;
};
```

This makes `**/folder`, `**/folder/`, and `**/folder/**` behave identically.

// Check if .git is a worktree reference file
const isGitWorktreeRef = async (gitPath: string): Promise<boolean> => {
  const stats = await fs.stat(gitPath);
  if (!stats.isFile()) return false;
  
  const content = await fs.readFile(gitPath, 'utf8');
  return content.startsWith('gitdir:');
};

// Adjust ignore patterns for worktrees
if (isWorktree) {
  // Remove '.git/**' pattern and add '.git' to ignore the reference file
  const gitIndex = adjustedIgnorePatterns.indexOf('.git/**');
  if (gitIndex !== -1) {
    adjustedIgnorePatterns.splice(gitIndex, 1);
    adjustedIgnorePatterns.push('.git');
  }
}

## File Size Limits

Files exceeding the size limit are automatically skipped.

```bash CLI
# Set maximum file size (default: 50MB)
repomix --input-max-file-size 10mb

# Supported units: kb, mb, gb
repomix --input-max-file-size 500kb
```

```json repomix.config.json
{
  "input": {
    "maxFileSize": 10485760  // 10MB in bytes
  }
}
```

**Implementation:** `src/core/file/fileRead.ts`

```typescript
const stats = await fs.stat(filePath);
if (stats.size > maxFileSize) {
  return {
    content: null,
    skippedReason: 'size_exceeded',
  };
}
```

## Binary File Detection

Binary files are automatically detected and excluded.

```typescript
// Check if file is binary by analyzing content
const isBinaryFile = (content: Buffer): boolean => {
  // Check for null bytes or high proportion of non-text characters
  for (let i = 0; i < Math.min(512, content.length); i++) {
    if (content[i] === 0) return true;
  }
  return false;
};
```

## stdin Mode

Process specific files from stdin:

```bash
# Process files from git diff
git diff --name-only | repomix --stdin

# Process files from find
find src -name "*.ts" -type f | repomix --stdin

# Process files from custom script
my-file-selector.sh | repomix --stdin
```

**Implementation:** `src/core/file/fileStdin.ts`

```typescript
export const readFilePathsFromStdin = async (cwd: string) => {
  const input = await readStdin();
  const filePaths = input
    .split('\n')
    .map(line => line.trim())
    .filter(line => line.length > 0)
    .map(line => path.resolve(cwd, line));
  
  return { filePaths };
};
```

<Note>
When using `--stdin`, don't specify directory arguments. File paths are read exclusively from stdin.
</Note>

## Empty Directories

Include empty directories in the directory tree:

```bash
# Include empty directories
repomix --include-empty-directories

# Show full directory structure (all dirs, even without included files)
repomix --include-full-directory-structure
```

**Implementation:** `src/core/file/fileSearch.ts:19`

```typescript
const findEmptyDirectories = async (
  rootDir: string,
  directories: string[],
  ignorePatterns: string[],
) => {
  const emptyDirs: string[] = [];
  
  for (const dir of directories) {
    const entries = await fs.readdir(fullPath);
    const hasVisibleContents = entries.some(entry => !entry.startsWith('.'));
    
    if (!hasVisibleContents) {
      const shouldIgnore = ignorePatterns.some(pattern => 
        minimatch(dir, pattern) || minimatch(`${dir}/`, pattern)
      );
      
      if (!shouldIgnore) {
        emptyDirs.push(dir);
      }
    }
  }
  
  return emptyDirs;
};
```
