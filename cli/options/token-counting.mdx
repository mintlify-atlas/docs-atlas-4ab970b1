Token counting options control how Repomix counts tokens for AI model context estimation.

## Token Count Encoding

<ParamField path="--token-count-encoding" type="string" default="o200k_base">
  Tokenizer model for counting tokens.

  ```bash
  # GPT-4o (default)
  repomix --token-count-encoding o200k_base

  # GPT-4/3.5
  repomix --token-count-encoding cl100k_base

  # GPT-3
  repomix --token-count-encoding p50k_base
  ```

  **Available encodings:**
  - `o200k_base` - GPT-4o, GPT-4o-mini (default)
  - `cl100k_base` - GPT-4, GPT-3.5-turbo, text-embedding-ada-002
  - `p50k_base` - GPT-3 (Davinci, Curie, Babbage, Ada)
  - `p50k_edit` - GPT-3 edit models
  - `r50k_base` - GPT-3 (older models)
</ParamField>

## Why Token Counting Matters

AI models have context limits measured in tokens:

- **GPT-4o** - 128,000 tokens
- **GPT-4-turbo** - 128,000 tokens  
- **GPT-3.5-turbo** - 16,385 tokens
- **Claude 3** - 200,000 tokens

Repomix counts tokens to help you:

1. **Estimate context usage** - Know if output fits model limits
2. **Optimize packing** - Remove unnecessary files
3. **Split intelligently** - Use `--split-output` with appropriate sizes
4. **Compare files** - See which files use most tokens

## Optimizing Token Usage

### 1. Remove Comments

```bash
repomix --remove-comments
```

Typically saves 10-30% tokens.

### 2. Remove Empty Lines

```bash
repomix --remove-empty-lines
```

Typically saves 5-15% tokens.

### 3. Compress Code

```bash
repomix --compress
```

Saves 50-80% tokens by keeping only structure.

### 4. Filter Files

```bash
# Only essential files
repomix --include "src/**/*.ts" --ignore "**/*.test.ts"
```

### 5. Combine Techniques

```bash
# Maximum optimization
repomix \
  --remove-comments \
  --remove-empty-lines \
  --compress \
  --include "src/**" \
  --ignore "**/*.test.*"
```

## Splitting for Context Limits

### Calculate Split Size

If your packed output is 100,000 tokens and model limit is 32,000:

```bash
# Split into ~30,000 token chunks
repomix --split-output 2mb --token-count-encoding o200k_base
```

**Rule of thumb:**
- 1 token ≈ 4 characters
- 32,000 tokens ≈ 128KB
- 100,000 tokens ≈ 400KB

### Model-Specific Splits

```bash
# GPT-3.5-turbo (16K context)
repomix --split-output 64kb --token-count-encoding cl100k_base

# GPT-4 (128K context)
repomix --split-output 512kb --token-count-encoding cl100k_base

# Claude (200K context)
repomix --split-output 800kb --token-count-encoding cl100k_base
```

## Advanced Usage

### Token Budget Workflow

```bash
#!/bin/bash
# pack-with-budget.sh

BUDGET=30000

# Initial pack
OUTPUT=$(repomix --token-count-encoding cl100k_base 2>&1)
TOKENS=$(echo "$OUTPUT" | grep "Total Tokens" | awk '{print $3}')

if [ "$TOKENS" -gt "$BUDGET" ]; then
  echo "Exceeds budget ($TOKENS > $BUDGET), optimizing..."
  
  repomix \
    --token-count-encoding cl100k_base \
    --remove-comments \
    --remove-empty-lines \
    --ignore "**/*.test.*"
    
  TOKENS=$(repomix 2>&1 | grep "Total Tokens" | awk '{print $3}')
  echo "Optimized: $TOKENS tokens"
fi
```

### CI Token Validation

```yaml
# .github/workflows/validate-tokens.yml
name: Validate Token Count

on: [pull_request]

jobs:
  check-tokens:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Pack and check tokens
        run: |
          OUTPUT=$(npx repomix --token-count-encoding cl100k_base 2>&1)
          TOKENS=$(echo "$OUTPUT" | grep "Total Tokens" | awk '{print $3}')
          
          if [ "$TOKENS" -gt 50000 ]; then
            echo "Error: Pack exceeds 50,000 token limit ($TOKENS tokens)"
            exit 1
          fi
          
          echo "✓ Token count OK: $TOKENS tokens"
```
